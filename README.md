# Using GitHub as a Data Backend for Your Dashboard

## Introduction
This guide is designed for non-developers who want to use GitHub as a data backend for their dashboard. By following these steps, you will be able to manage your data efficiently and effectively.

## Prerequisites
- A GitHub account
- Basic understanding of how to navigate GitHub

## Step 1: Create a Repository
1. Log in to your GitHub account.
2. Click on the `+` icon in the top right corner and select `New repository`.
3. Name your repository (e.g., `DashboardData`).
4. Add a description if desired.
5. Choose `Public` or `Private` for your repository.
6. Click `Create repository`.

## Step 2: Adding Data Files
1. Navigate to your repository on GitHub.
2. Click on `Add file` and then `Upload files`.
3. Drag and drop your data files or click `choose your files` to select them.
4. Add a commit message (e.g., `Adding initial data files`).
5. Click `Commit changes`.

## Step 3: Updating Data Files
1. Go to the file you want to update in your repository.
2. Click on the file name.
3. Click the pencil icon to edit the file directly in the browser or upload new versions of the file.
4. Add a commit message describing your changes.
5. Click `Commit changes`.

## Step 4: Using Raw Files in Your Dashboard
1. Navigate to the file in your repository.
2. Click on the `Raw` button to view the raw data file.
3. Copy the URL from the address bar. You will use this URL in your code to fetch the data.

## Step 5: Regularly Updating Your Data
- Remember to repeat the steps in "Adding Data Files" and "Updating Data Files" as needed to keep your data current.

## Conclusion
By following these instructions, you can easily use GitHub as a data backend for your dashboard. Keep your data organized, updated, and accessible with GitHub!
